. تنظیم نام و ایمیل (یک‌بار برای هر سیستم)

Git نیاز دارد بداند چه کسی تغییرات را اعمال کرده. در سیستم دوم (یا هر سیستم جدید) این را اجرا کن:

git config --global user.name "Mehdi Shakiba"
git config --global user.email "your-email@example.com"


این فقط یک‌بار کافی است، اگر بخواهی فقط برای یک پروژه خاص باشه، --global را حذف کن.

۲. دریافت آخرین تغییرات از 
قبل از شروع کار روی سیستم دوم، همیشه ابتدا تغییرات را از 
دریافت کن تا پروژه به‌روز باشد:

git pull origin main

اگر شاخه‌ی اصلی master است، به جای main بنویس master.


۳. افزودن تغییرات جدید به 
بعد از اینکه فایل‌ها را تغییر دادی یا فایل جدید اضافه کردی:

git add .
git commit -m "1404.09.121"

git add . تمام فایل‌های تغییر کرده را آماده می‌کند.
git commit -m "..." تغییرات را ثبت می‌کند.

۴. ارسال تغییرات به 
برای فرستادن تغییرات روی ریپازیتوری آنلاین:

git push origin main

دوباره اگر شاخه اصلی master است، main را با master جایگزین کن.

⚠️ از آگوست 2021، برای HTTPS باید Personal Access Token (PAT) را به جای پسورد وارد کنی.

۵. بررسی وضعیت (اختیاری ولی مفید)
git status


می‌بینی چه فایل‌هایی تغییر کرده، آماده commit هستند یا نه.

همیشه قبل از commit و push مفید است.
